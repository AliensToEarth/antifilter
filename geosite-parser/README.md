# GeoSite Parser Utility

This utility processes geosite data using the `geosite-antifilter.json` configuration file. It runs automatically after the `make geosite` command to create filtered geosite output files.

## Purpose

The GeoSite Parser reads the generated `geosite.dat` file from the main geosite build process and creates a filtered version (`geoip-antifilter.dat`) containing only the specified domain lists.

## Configuration

The utility uses the `geosite-antifilter.json` configuration file located in the project root. This configuration defines:

- **Input**: Reads from `../output/geosite.dat` (generated by the main geosite build)
- **Output**: Creates `geoip-antifilter.dat` with filtered domain lists

### Current Configuration

```json
{
    "input": [
        {
            "type": "readGeoSiteDat",
            "action": "add",
            "args": {
                "path": "../output/geosite.dat"
            }
        }
    ],
    "output": [
        {
            "type": "v2rayGeoSiteDat",
            "action": "output",
            "args": {
                "outputDir": "../output",
                "outputName": "geoip-antifilter.dat",
                "wantedList": ["google", "youtube", "meta", "antifilter-community", "private"]
            }
        }
    ]
}
```

## Usage

### Automatic Usage (Recommended)

The utility runs automatically as part of the geosite workflow:

```bash
make geosite
```

This will:
1. Download antifilter community domains
2. Build the main geosite.dat file
3. **Automatically run the parser** to create the filtered output

### Manual Usage

You can also run the parser manually:

```bash
cd geosite-parser
go run main.go
```

Or with a custom config file:

```bash
cd geosite-parser
go run main.go -c /path/to/custom-config.json
```

### Help

```bash
cd geosite-parser
go run main.go -h
```

## Output

The utility creates:
- `../output/geoip-antifilter.dat` - Filtered geosite data containing only the specified domain lists

## Dependencies

- Go 1.21 or later
- `github.com/v2fly/geoip` library (automatically downloaded via `go mod tidy`)

## Integration with Makefile

The utility is integrated into the main Makefile workflow:

- **Target**: `geosite-parse`
- **Dependencies**: Runs after `geosite-build`
- **Automatic**: Included in the main `geosite` target
- **Cleanup**: Included in `make clean`

## Troubleshooting

1. **Missing input file**: Ensure `make geosite-build` has run successfully and created `../output/geosite.dat`
2. **Go module issues**: The Makefile automatically runs `go mod tidy` before execution
3. **Permission errors**: Ensure the `../output` directory is writable
4. **Config file not found**: Verify `geosite-antifilter.json` exists in the project root
